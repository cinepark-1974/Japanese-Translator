# 🎌 Japanese-Translator v1.0

### BLUE JEANS PICTURES — 5-Stage Market Adaptation Pipeline

한국어 시나리오를 **일본 프로듀서가 "우리 관객한테 통한다"고 느끼는 일본어 시나리오**로 변환하는 AI 번역 엔진.  
단순 번역이 아닌, **문화코드 변환 → 경어 설계 → 시나리오 포맷 변환 → 대사 현지화 → 품질 검증**까지 5단계 파이프라인을 자동 실행합니다.

> **언어 변환이 아닌 시장 변환(Market Adaptation)**

---

## 🚀 핵심 특징

### 5-Stage Pipeline

| Stage | 이름 | 모델 | 역할 |
|-------|------|------|------|
| **1** | Raw Translation | Sonnet | 한국어 → 일본어 충실한 직역 + 캐릭터명/장소명/통화 매핑 |
| **2** | Format Conversion | 규칙 기반 | 한국 시나리오 포맷 → 일본 〇柱/ト書き/「」セリフ 변환 |
| **3** | Voice Rewrite | Opus | 번역체 제거, 일본 시나리오 문체로 리라이팅 |
| **4** | Dialogue Polish | Opus | 경어 설계 적용, 캐릭터별 대사 현지화 |
| **5** | QA Check | Sonnet | 포맷/경어/문화코드/스토리 검증 + 점수화 리포트 |

### English Translator Engine과의 차이

| 항목 | English Translator | Japanese Translator |
|------|-------------------|---------------------|
| 핵심 과제 | 언어 변환 | **시장 변환** |
| 호칭 체계 | 평탄화 (first name) | **5단계 경어 설계** |
| 문화코드 | 등가물 치환 | **전승 연결 (물귀신→水姫)** |
| 포맷 | INT./EXT. Slug Line | **〇柱 / ト書き3字下げ / 「」** |
| 출력 | US Letter / A4 횡서 | **A4 横書き (기획모드)** |
| 통화 | 직역 또는 USD | **1/10 고정 변환 (원→엔)** |

### 경어 톤 태그 (5단계)

| 태그 | 대상 | 특징 |
|------|------|------|
| `keigo` | 비즈니스, 경찰, 공식 관계 | です/ます + 敬語 |
| `teinei` | 초면, 서비스업, 일반적 거리감 | です/ます, 부드러운 톤 |
| `tameguchi` | 동등 관계, 친구, 독백 | だ/する, 축약형 |
| `kenson` | 위압적 상대에게 쓰는 겸양어 | いたします, 존경+겸양 |
| `ibar` | 갑질, 위에서 내려다보는 말투 | タメ口 + 명령형 |

## ⚙️ 설치 및 실행

### 로컬 실행

```bash
# 1. 클론
git clone https://github.com/cinepark-1974/japanese-translator.git
cd japanese-translator

# 2. 의존성 설치
pip install -r requirements.txt

# 3. API 키 설정
mkdir -p .streamlit
echo 'ANTHROPIC_API_KEY = "sk-ant-your-key-here"' > .streamlit/secrets.toml

# 4. 실행
streamlit run main.py
```

### Streamlit Cloud 배포

1. GitHub에 push
2. [share.streamlit.io](https://share.streamlit.io) 에서 앱 생성
3. Secrets에 `ANTHROPIC_API_KEY` 추가
4. 배포 완료

## 🔧 번역체 안티패턴 룰셋 (Stage 3)

Voice Rewrite 단계에서 자동으로 감지·교정하는 일본어 번역체 패턴 9가지:

| 코드 | 이름 | Bad → Good |
|------|------|------------|
| JP-1 | 한국어 어순 직역 | "見ている窓の外を" → "窓の外——貯水池を見つめる" |
| JP-2 | 「〜している」남용 | "確認している。ついている" → "確認する。ため息" |
| JP-3 | 감정 직접 서술 | "怒りを感じた" → "拳が白くなる" |
| JP-4 | 설명적 지문 | "落ち込んでいることを示す" → "未開封の郵便物。止まった時計" |
| JP-5 | セリフ 문말 단조 | 「ます」연속 → バリエーション |
| JP-6 | 의성어 직역 | 한국 의성어 → 일본어 등가물 |
| JP-7 | 비트 중복 | "立ち止まる。間。静寂" → "間。" |
| JP-8 | 지문 3행 초과 | 분할하여 여백으로 템포 생성 |
| JP-9 | 문화적 직역 | 한국 정서 코드 → 일본 관객 등가물 |

## 📊 듀얼 모델 정책

- **Sonnet**: 번역(Stage 1), QA(Stage 5) — 정확도와 속도의 균형
- **Opus**: Voice Rewrite(Stage 3), Dialogue Polish(Stage 4) — 문화적 뉘앙스와 경어 설계에 최고 품질 필수

Stage 2(Format Conversion)는 규칙 기반이므로 API 호출 없이 즉시 처리됩니다.

## 📝 출력 포맷

### 企画モード (横書き) — 투자제안/피칭용

| 항목 | 사양 |
|------|------|
| 용지 | A4 (210×297mm) |
| 방향 | 横書き (가로쓰기) |
| 폰트 | Yu Gothic (ト書き) / Yu Mincho (타이틀) |
| 씬 헤더 | 〇場所（時間帯） |
| ト書き | 3字下げ、現在形 |
| セリフ | 人物名 行頭、「」カギカッコ、文末句点なし |
| 출력 | DOCX + TXT 동시 |

> 촬영대본용 종서(縦書き 20字×20行) 포맷은 일본 공동제작사의 영역이므로 본 엔진의 범위에 포함하지 않습니다.
